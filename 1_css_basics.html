First Steps

  • A nice guide to CSS Selectors: Mozilla Developer Network
  • HTML5 is for defining "semantic content".
  • CSS3 is for visual design: fonts, colors, and layout.
  • (demo of fonts and colors)
  • span and div are used as grouping constructs so you can style a set of elements as a contiguous group.
  • Browser-specific defaults gave rise to the invention of reset stylesheets to provide a known starting point regardless of browser.
  • Two good resets: Eric Meyer's Reset and Normalize.

Visual Layout and Spacing

  • Layout: every element is a rectangle.
  • There are three primary values for the display property: inline, block, inline-block. There are others that are sometimes handy, too.
  • How do we know how much room each element will occupy on the screen? It's hard to know. The CSS box model is pretty complicated.
  • Multi-column layouts require near-expert understanding of CSS.
  • Mobile devices often require custom layouts in order to be touch-friendly and font-size-friendly.
  • A nice guide to Media Queries: Mozilla Developer Network

CSS Frameworks

  • CSS Frameworks are just files of CSS rules that someone else wrote.
  • Most CSS frameworks are nothing more than (large) set of CSS rules that define a (large) set of CSS classes.
  • Therefore, if you use a CSS framework, your work is nicely reduced to simply attaching the appropriate CSS classes onto your HTML elements by using the class attribute.
  • One of the best features of good CSS framework is a grid system that lets you build multi-column layouts.
  • I think Bootstrap is a nice CSS framework to learn first.
